# Container Throughput Results - 2026-07-09

Status: frozen historical measurement

These numbers describe the named toolchains and the 2026-07-09 compiler only.
The dictionary implementation has since gained grouped control-byte probing,
tombstones, generic key/value mutation, and self-host parity. Re-run the
benchmark suite before comparing the current compiler.

Machine-local comparison between Sollang native output, C#/.NET, Go, and Rust on the same workload.

Commands:

```powershell
.\scripts\sollang.ps1 -Source benchmarks\containers-throughput.slg -Output artifacts\benchmarks\containers-throughput.exe -KeepTemps
dotnet build benchmarks\csharp\ContainersThroughput\ContainersThroughput.csproj -c Release --nologo
Push-Location benchmarks\go\containers-throughput
& 'C:\Program Files\Go\bin\go.exe' build -o ..\..\..\artifacts\benchmarks\go-containers-throughput.exe .
Pop-Location
& "$env:USERPROFILE\.cargo\bin\cargo.exe" build --release --manifest-path benchmarks\rust\containers-throughput\Cargo.toml
1..5 | ForEach-Object { .\artifacts\benchmarks\containers-throughput.exe }
1..5 | ForEach-Object { dotnet run --project benchmarks\csharp\ContainersThroughput\ContainersThroughput.csproj -c Release --no-build }
1..5 | ForEach-Object { .\artifacts\benchmarks\go-containers-throughput.exe }
1..5 | ForEach-Object { .\benchmarks\rust\containers-throughput\target\release\containers-throughput.exe hashbrown }
1..3 | ForEach-Object { .\benchmarks\rust\containers-throughput\target\release\containers-throughput.exe std }
```

Toolchains:

- C#: .NET `11.0.100-preview.5.26302.115`
- Go: `go1.26.5 windows/amd64`
- Rust: `rustc 1.97.0`
- Rust fast map: `hashbrown 0.16.1`, using its default `foldhash` hasher

Large workload:

- Dynamic array build: append `1..50_000_000`.
- Dynamic array scan: sum the array 20 times.
- Dictionary build: insert keys `0..8_000_000`.
- Dictionary lookup: sum values for keys `1..8_000_000`.

Tested operations:

- Array build: create an empty growable integer array/list/vector/slice, then append integers from `1` to `50,000,000`.
- Array scan: iterate the completed array sequentially and sum all values, repeated `20` times.
- Dictionary build: create an integer-to-integer dictionary/map/hash map with initial key `0`, then insert keys `1` to `8,000,000` with value `key * 3`.
- Dictionary lookup: look up keys `1` to `8,000,000` from the completed dictionary/map/hash map and sum the returned values.
- Correctness guards: `arrayChecksum` must be `25,000,000,500,000,000`, and `dictChecksum` must be `96,000,012,000,000`.
- Timing rule: build time and scan/lookup time are measured separately inside each program; the main comparison uses five runs and reports the median.

Rust's standard `HashMap` default hasher is currently SipHash 1-3. That default is security oriented and can be much slower for small integer keys than faster non-cryptographic hashers. The 4-way comparison above therefore uses `hashbrown` for the Rust performance-oriented row and keeps `std HashMap` as a separate reference.

Memory notes:

- Sollang, C#, and Rust array final backing bytes are `536,870,912`; Go array final backing bytes were `469,999,616` in the observed run.
- Sollang dictionary final backing bytes: `285,212,672`.
- C# dictionary build allocated bytes reported by GC: `660,410,488`.
- Go dictionary build allocated bytes reported by runtime stats: about `604 MB` in the observed runs.
- Go map capacity/backing bytes and Rust allocation bytes are not exposed by these simple benchmark programs.
- These memory numbers are not identical metrics. Sollang currently reports estimated live backing storage; C#, Go, and Rust expose different runtime allocation/capacity information.

Optimization delta:

- Array scan improved from `453 ms` to `344 ms`.
- Dictionary build improved from `422 ms` to `188 ms`.
- Dictionary lookup improved from `234 ms` to `94 ms`.
- Applied changes: power-of-two bitmask probing, phi-based find result, H2 reuse, division-free entry offset calculation, a cheaper integer-specialized multiplicative hash, lookup-only probing, LLVM memset for dictionary control bytes, native `-O3`, and 7/8 grow threshold.

Interpretation:

- The current Sollang dynamic array path is already in a reasonable range against C# `List<long>` for this simple append/scan benchmark. Array build is essentially tied; array scan is faster in this run.
- The earlier Rust result used `std::collections::HashMap`, so it measured Rust's security-oriented default hasher more than the fastest Rust map layout. With `hashbrown`, Rust dictionary lookup is close to Sollang in this workload.
- The dictionary path improved materially. In this workload, Sollang is faster than Go's default map and Rust `hashbrown`, but still slower than C# `Dictionary<long,long>`, especially lookup.
- Future benchmark work should add Sollang allocator counters and expose the native optimization mode explicitly instead of relying on the current Windows linker default.
